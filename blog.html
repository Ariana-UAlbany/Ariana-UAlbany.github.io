Strategy Design Pattern Presentation

October 10, 2025

What is the problem that thee Strategy Design Pattern aims to solve and how?

Behavioral Issues arise when the same job can be done in many different ways. Code can become:
  1. BLOATED The more behaviors exist to complete a task, the larger the code can get. This causes an issue with scalability because as code gets bigger, it gets harder to read, test and maintain.
  2. HARD TO EXTEND Open/Closed means that code should be open to addition, but closed for modification. Adding new behaviors means modifying the main conditional each time which risks breaking existing logic.
  3. PRONE TO MERGE CONFLICTS because there is a higher chance for developers to be working on the same code block when modifying behaviors

Designing an Operating System (OS) Simulator

October 6, 2025

Simulate cooperative multitasking operating system using Java

UPDATE: Added priority scheduler

On startup, the OS creates processes to run. When a process' quantum expires, the static Kernel object takes control and switches to another process waiting in queue. In this update, the queue is divided into three different queues based on priority. A process that starts as realtime gets demoted to interactive if it reaches timeout several times in succession. An interactive process then demotes to background. Switching a process selects a priority queue using a probabilistic model. Cooperation is facilitated via a binary semaphore.
